# 操作系统

操作系统（Operating System, OS）是连接硬件与应用程序的桥梁，是计算机系统中最核心的软件之一。它负责资源管理（CPU、内存、磁盘、网络等）以及进程调度、文件系统、安全控制等功能。

它在计算机学科“四大基础”中和数据结构、组成原理、计算机网络并列，也是考研 408 的必考科目。在大厂面试中，操作系统的知识考察频率要明显高于计算机组成原理，尤其是多线程、内存管理、锁机制、系统调用等内容。

## 学习重点

建议重点掌握以下内容（也是考研与面试的高频点）：

1. 操作系统概述

    - 操作系统功能与分类

    - 系统调用、内核态与用户态

2. 进程与线程

    - 进程/线程的创建、终止、状态切换

    - 调度算法（FCFS、SJF、RR、多级反馈队列）

    - 线程模型（用户级 / 内核级线程）

    - 上下文切换

3. 进程间通信（IPC）与同步

    - 管道、消息队列、共享内存、信号量

    - 互斥、同步、死锁与死锁处理方法

4. CPU 调度与同步原语

    - 信号量（P/V 操作）

    - 互斥锁、自旋锁、条件变量

5. 内存管理

    - 连续分配与非连续分配

    - 分页、分段、段页式存储

    - 虚拟内存、页面置换算法（FIFO、LRU、OPT）

6. 文件系统

    - 文件控制块（FCB）、索引节点（inode）

    - 文件访问控制

    - 磁盘调度算法（FCFS、SSTF、SCAN、C-SCAN）

7. 设备管理与 I/O

    - 缓冲区、缓存一致性

    - 中断处理

    - DMA（直接内存访问）

8. 系统安全与虚拟化（扩展）

    - 权限管理

    - 虚拟机与容器技术的 OS 原理

## 书籍推荐

1. 《深入理解计算机系统》（CSAPP）

尤其是进程、虚拟内存、异常控制流等章节，与 OS 内容高度相关。适合配合组成原理一起学，打好底层运行机制基础。

2. 《Operating Systems: Three Easy Pieces》（OSTEP）

语言通俗易懂，用最通俗的语言讲最硬核的知识，莫过于这本书。整本书章节设计围绕 OS 三大核心主题：虚拟化、并发、持久化，且每章后面有编程实验（Pintos、xv6、自己实现简单调度器等），强烈阅读。

## 课程推荐

1. NJU Operating System: Design and Implementation

推荐南京大学的“操作系统”课程，也就是蒋炎岩老师的课程。JYY作为较年轻的老师，其编程经验也是非常丰富，经常在课上一言不合就打开控制台准备编码，相较于大部分只知理论而不知实践的老师，不知道强了多少。

这门课对于程序的观点也非常有趣，“程序就是状态机”这一视角非常新颖，也借着这个视角引入了并发控制的常见方法。

[南京大学 操作系统：设计与实现](https://www.bilibili.com/video/BV1Cm4y1d7Ur/)

2. MIT 6.S081 Operating System Engineering

唯一真神，伟大无需多言。个人认为是最强操作系统课程，但是内容相对来讲比较硬核，不太容易看懂，且配套Project的难度相当大，需自行取舍。

[MIT 6.S081 Operating System Engineering](https://www.bilibili.com/video/BV1rS4y1n7y1)

## 小贴士

- 学 OS 不只是看概念，一定要动手写代码，哪怕是小实验。

- 进程调度、虚拟内存、锁这些抽象概念，用调试器（gdb）和打印信息观察效果会更直观。

- 如果时间有限，建议先掌握进程与线程、内存管理、同步机制，这些是面试和考试的高频内容。
