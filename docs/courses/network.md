# 计算机网络

计算机网络是计算机专业“四大件”之一（数据结构与算法、计算机组成原理、操作系统、计算机网络），无论是考研 408，还是前端/后端开发、大厂面试，它的地位都非常稳固：**你可以忘记路由器长什么样，但不能不知道 TCP 为什么要三次握手，以及从键入网址到网页显示过程都发生了什么**。

相比“有点抽象”的组成原理，计算机网络的内容更贴近日常，理解难度整体更低；但是——内容多、细节杂、协议极其爱考你边边角角、需要记忆的内容非常多，所以别掉以轻心。

如果你以后想从事前端、后端、网络、安全等方向，计算机网络这门课绝对是底座。

## 学习重点

目前主流的学习顺序有“从上往下”和“从下往上”两种方式，对应的教材分别是 Jame F.Kurose 的《计算机网络 自顶向下方法》和 谢希仁的《计算机网络》。笔者个人更加推荐采用“从上往下”的方式，因为上层是大家更普遍接触到的应用层，相对来讲学习起来会比较轻松，也能够更快建立对网络的初步理解。

1. 应用层

    - HTTP/1.1、HTTP/2、HTTP/3 的区别

    - DNS 查询流程、缓存机制

    - 常见的应用协议（FTP、SMTP、DHCP）

    - HTTPS：TLS 握手、对称加密 vs 非对称加密

2. 运输层（TCP协议是核心）

    - TCP 三次握手与四次挥手

    - 滑动窗口、流量控制、拥塞控制

    - UDP 特点与使用场景（DNS、流媒体、实时游戏）

    - QUIC 协议的核心思想（基于 UDP + TLS 1.3）

3. 网络层

    - IP 地址、CIDR、子网划分

    - ARP、ICMP（别忘了 ping 为什么能 ping 通）

    - 路由协议（RIP / OSPF / BGP）

4. 数据链路层

    - 以太网帧结构

    - MAC 地址、交换机的工作原理

    - VLAN、链路聚合

5. 物理层

    - 信号、带宽、噪声

    - 编码方式（NRZ、曼彻斯特编码）

    - 这个层考研会考，但实操基本不会直接用到

## 书籍推荐

1. 《计算机网络 自顶向下》

几乎是国内 CS 学生最容易看懂、结构最清晰的网络教材。从应用层讲起，讲得既轻松又透彻。

2. 《网络是怎样连接的》

用最通俗易懂的语言讲解最硬核的网络知识，语言简单且幽默风趣，三岁小孩都能看懂，推荐作为计算机网络的入门教材。

3. 《图解HTTP》和《图解TCP_IP》

合称“图解系列”。学习过计算机网络的同学都应该明白计算机网络的知识相当琐碎且不好记忆，那么有没有一种更加简单、有趣的方式来帮助我们学习、记忆计算机网络的知识呢？当然有，就是“图解系列”的这两本书。这套书使用了大量的图画，用有趣的图画给读者传递计算机网络的知识，相信看完这套书之后一定能对计算机网络会有更深的理解。
